Regular Expressions

The following regular expressions can be used in the search pattern when renaming or cloning multiple files and the regular expressions option is enabled:

Expression Description
. Matches any single character.
\ Escapes the character following the backslash.
\xxxxx Matches the Unicode character corresponding to the hexadecimal number hhhh.
\0ooo Matches the ASCII character corresponding to the octal number ooo.
\d Matches a digit.
\D Matches a non-digit.
\s Matches a whitespace character.
\S Matches a non-whitespace character.
\w Matches a word character (a letter, digit or undescore).
\W Matches a non-word character.
[...] Matches any character listed in the set. Within the set a dash can be used to indicate a range of characters, e.g. [0-9] matches any digit.
[^...] Matches any character not listed in the set.
| Matches either expression it separates.
E? Matches zero or one occurrences of E (where E is a single character, set of characters or group).
E* Matches zero or more occurrences of E.
E+ Matches one or more occurrences of E.
E{n} Matches exactly n occurrences of E.
E{n,} Matches at least n occurrences of E.
E{,m} Matches at most m occurrences of E.
E{n,m} Matches at least n and at most m occurrences of E.
(...) Groups a subexpression together so it can be quantified and captured.
\1, \2, ... Backreference; matches the same text as previously matched within first, second, etc., set of parentheses. It can also be used in the replacement text to insert a captured group.
^ Matches only at the beginning of the string.
$ Matches only at the end of the string.
\b Matches only at a word boundary (the beginning or the end of a word).
\B Matches only at a non-word boundary.
(?=E) Positive lookahead; matches only if E matches at the current position.
(?!E) Negative lookahead; matches only if E does not match at the current position.
(?:E) Non-capturing group.
